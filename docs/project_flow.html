End-to-End Prediction Flow

📥
1 · Request Received
Client sends a POST to /predict with customer features and the X-API-Key header. FastAPI validates the payload against the Pydantic schema — any missing or invalid field returns 422 immediately.
POST /predict X-API-Key auth Pydantic v2
🔒
2 · Auth & Dependency Injection
FastAPI's Depends(verify_api_key) checks the key against the comma-separated VALID_API_KEYS environment variable. The DB session is also injected via Depends(get_db), ensuring every request gets its own session that's automatically closed.
Depends() Session lifecycle Env-var keys
🤖
3 · ML Inference
The sklearn Pipeline (StandardScaler → OrdinalEncoder → RandomForest) runs prediction in a single call — no separate preprocessing step. The model outputs both a binary label and a probability, which is translated to a risk tier (High ≥ 0.7, Medium ≥ 0.4, Low below).
RandomForest sklearn Pipeline class_weight='balanced'
💾
4 · Logged to SQLite
Every prediction is logged to the prediction_logs table via SQLAlchemy ORM — storing the input features (as JSON), prediction, probability, risk label, and UTC timestamp. This powers the monitoring dashboard.
SQLAlchemy ORM SQLite JSON features
📤
5 · Response Returned
A structured JSON response is returned: prediction (0/1), probability (0–1 float), risk_label, and an explanation string. The Streamlit UI renders this as a gauge chart with color-coded risk.
JSON response Gauge chart Risk tier

Key Features

⚡
Single Prediction
Real-time churn scoring for one customer. Sub-millisecond inference using a local sklearn pipeline — no external API calls, no latency.
📦
Async Batch Scoring
Upload a CSV of customers. A background thread scores them asynchronously while you poll for status. Download the result CSV when done.
📈
PSI Drift Monitoring
Population Stability Index computed over a rolling window for tenure, monthly charges, and total charges. Alerts when live data drifts from training.
🤖
RAG Knowledge Assistant
Ask questions about churn drivers, model internals, or API usage. FAISS + TF-IDF retrieval over 3 knowledge docs — fully offline.
🔐
API Key Auth
Stateless authentication via X-API-Key header. Keys are read from environment variables — no database, no user management, no session state.
🐳
Dockerized & Cloud-Ready
Multi-stage Dockerfile with non-root user, health check, and volume mounts. Deployed on AWS EC2 via Docker Hub image pull.

Design Decision: The model is tuned for high recall using class_weight='balanced'. In churn, it's better to flag a loyal customer as at-risk (false positive) than to miss a real churner (false negative). Missing a churner costs a retention offer; a false alarm costs a courtesy call.

Async Batch Scoring Flow

📤
Upload CSV
Client uploads a CSV file via POST /batch. FastAPI creates a BatchJob row in SQLite with status pending and returns a job_id immediately — no waiting for scoring to finish.
job_id returned instantly
⚙️
Background Thread Scores
A daemon thread picks up the job, opens its own DB session (thread-safe), and runs ml_service.predict_batch() row by row. Each prediction is logged to prediction_logs for monitoring.
threading.SemaphoreThread-safe DB
🔄
Poll for Status
Client polls GET /batch/{job_id} until status becomes done or failed. The Streamlit UI does this automatically with a spinner loop.
pending → running → done
⬇️
Download Results
Scored CSV is saved to data/batch_results/. Client downloads via GET /batch/{job_id}/download — the scored CSV includes all original columns plus prediction, probability, and risk_label.
StreamingResponseCSV download
